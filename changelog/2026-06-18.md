# Release — 2026-06-18 (pending)

_Changes since the previous release (commit `4d74ee6`)._

## Added
- **New Task — scan to link asset (iOS/Android):** Creating a task on mobile now
  opens the camera to scan the asset's QR instead of searching a list. The
  scanned asset is validated (must exist and have an asset assigned) before it's
  linked; web still uses text search.
- **New Asset — address autocomplete:** Location fields now show Google Places
  address suggestions as you type (pick one to fill the field).
- **Swap QR — unified search:** A single "Search for an asset" box that matches
  on name, type, model, serial, asset ID, or assignee — shown above
  "Find by Asset ID" — replacing the separate Model / Type / Assigned fields.
- **End of Life — restore:** Admins now get a **"Bring back in service"** action
  when scanning an End-of-Life asset.

## Changed
- **Standardised "Clear all" across web filters:** Activity, Search, and Documents
  filter dialogs now all use the same **"Clear all"** wording and the same
  shown-but-disabled-when-empty behaviour. (Activity gained the button; Search now
  shows it disabled instead of hidden; Documents' "Clear" was renamed to "Clear
  all" and disabled when empty.) Sort dialogs always carry a default, so there's
  nothing to clear there.
- **Quick-action available to all users (web search table):** The per-row ⚡
  quick-action button (opens an asset's actions) is no longer admin-only — every
  web user now gets it. The bulk-select checkbox column stays admin-only.
- **Edit button width (non-admin):** On the asset and asset-type screens, when a
  user only sees the **Edit** action (Copy/Delete are admin-only), the button no
  longer stretches the full bar — it's now a sensibly sized, left-aligned button.
- **Web asset list (master-detail):** In the preview pane you can now click the
  asset's image or title/ID to open the full asset page — you no longer have to
  use the "Open full" button. A "Click to open full details ↗" hint was added.
- **Quick-action buttons (check-in screen):** Colour is now purely positional in
  every scenario (In Service, Placeholder, QR-reserved, awaiting-QR, End of
  Life) — the **left** column is always blue (filled) and the **right** column
  always orange (outlined), regardless of which action lands there.
- **Imported-asset quick actions:** Standardised the button grid — blue actions
  on the left, orange on the right — and "Assign Imported Asset" now stays on
  one line.
- **Places API:** The address-autocomplete endpoint now reports the real Google
  error (e.g. `REQUEST_DENIED`) instead of silently returning no results, so
  misconfiguration is visible.

## Fixed
- **iOS date picker:** Spinner text was washed out / unreadable — now
  high-contrast (forced light theme + explicit text colour).
- **New Asset (scan to select ID):** Form fields appeared ~1 second after
  scanning a QR; they now appear immediately.
- **QR-reserved assets:** The "Add image" button is hidden until a real asset is
  assigned to the QR.
- **Swap via Scan QR:** Scanning an empty / reserved / End-of-Life QR (or the
  same QR) is now blocked with a clear validation message instead of proceeding.
- **End-of-Life screen:** The "Back to Dashboard" button no longer truncates to
  "Back to …".
