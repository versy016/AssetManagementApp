# Changelog

Release notes for GearOps — **one file per release**, newest by date.
Each file lists what changed, grouped into **Added**, **Changed**, **Fixed**
(Keep a Changelog style).

## Convention
- While preparing the next release, record changes in a dated file
  (e.g. `2026-06-18.md`) with the heading marked **(pending)**.
- On release, set the version + date in that file's heading and add a one-line
  entry to the index in [`/CHANGELOG.md`](../CHANGELOG.md).
- Start a **new file** for the next release.

The repo-root [`CHANGELOG.md`](../CHANGELOG.md) is the index of all releases.
