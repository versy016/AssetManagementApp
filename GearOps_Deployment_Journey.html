Phase 02
Pilot Migration
50 assets. Learn everything before the full rollout.
⏱ 1 Week
▾

What happens

📋
Select 50 representative assets from Go Codes as the pilot batch
⬆️
Import those 50 asset records into GearOps via migration script
🏷️
Remove old Go Codes stickers, apply new GearOps QR stickers
📷
Scan every new sticker — confirm 100% resolve to correct records
📝
Document any friction, errors, or process improvements found

Why a pilot first?

⚡
Catch data mapping issues before they affect 500 assets
🎯
Train field techs on the sticker replacement procedure hands-on
⏱️
Measure time per asset — helps forecast the full migration window
🛡️
Zero risk to the broader fleet — issues stay contained to 50 assets

✓ Gate to Phase 3 — Pilot must confirm:

✔ All 50 pilot assets scanned and verified in GearOps

✔ Data import row count matches Go Codes source
✔ Project Lead sign-off documented

Physical Migration

Out with the old sticker.
In with the new.

Every asset gets a GearOps QR sticker. The process is simple, and each one is verified on the spot.

🏷️

Go Codes Sticker

Legacy barcode label.
Old system, old data.

RETIRING
→
⬛

GearOps QR Sticker

New QR code. Scans directly into the app on iOS, Android or web camera.

REPLACING
📷

Photograph the old sticker

Create an audit trail before removing anything. Photo stored per asset.

🧹

Clean the surface

Isopropyl wipe, let dry. New sticker needs a clean bond to last in the field.

🏷️

Apply GearOps sticker

Peel, align, press firmly. Each sticker pre-printed with the correct asset ID.

📱

Scan to verify

Open GearOps, scan the new sticker. Confirm the right asset record loads.

✅

Mark complete

Tick off in the migration sheet. Move to the next asset.

Risk Register

What could slow us down —
and how we prevent it.

Every risk has an owner and a plan. These are reviewed at each phase gate.

Assets not fully catalogued in Go Codes
High
Audit all Go Codes records before migration. Reconcile any discrepancies against physical asset list before Phase 3.
📌 Owner: Project Lead
Supabase migration breaks production
High
All migrations tested on staging first. Supabase backup taken before every push. Rollback script prepared and tested.
📌 Owner: Dev / IT
Data loss during Go Codes CSV export
High
Full CSV backed up to secure storage immediately after export. Row count validated before any import begins.
📌 Owner: Dev / IT
iOS App Store review delays
Medium
Submit the production build at least 5 days before target go-live. TestFlight remains the fallback for internal teams during review.
📌 Owner: Dev / IT
QR sticker misapplied or damaged
Medium
Print 10% spare stickers for every batch. Scan verification immediately after application. Any failed scan → replace on the spot.
📌 Owner: Field Tech Lead
Field staff resist the new system
Medium
Pilot group becomes champions. Hands-on training session before Phase 3. Quick-reference cheat sheet printed and in their kit.
📌 Owner: Office Manager
